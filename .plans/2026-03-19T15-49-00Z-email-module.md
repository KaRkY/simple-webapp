assumptions:
- Template body stored in DB as Thymeleaf template string; rendered via StringTemplateResolver (not classpath files)
- UserRegisteredEvent gets activationToken field added
- Email module queries user module via public UserManagement.findById(UUID) for recipient email
- emails table is a queue; sent emails move to emails_archive
- EmailManagementController manages templates (CRUD) via admin UI — left as stub for now
- spring.mail.from configured in application.properties
