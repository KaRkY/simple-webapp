assumptions:
- / must be publicly accessible (no redirect to /login).
- Unauthenticated users see: login form + link to /register.
- Authenticated users see: welcome message, link to /admin/users (ADMIN only, via th:if),
  link to /account/change-password, logout button.
- Existing /login page is kept (Spring Security still uses it for form processing endpoint).
- defaultSuccessUrl changes from "/" stay as-is (login still redirects to /).
- Spring Security's formLogin.loginPage("/login") stays — form POSTs to /login.
  The home page login form also POSTs to /login.
- No new controller needed — a simple @Controller with @GetMapping("/") returning "index".
  OR use SimpleUrlHandlerMapping / mvc:view-controller if no logic needed.
  Use a @Controller for clarity.
- Admin link visibility: Thymeleaf Security extras dialect
  (sec:authorize="hasRole('ADMIN')") — already on classpath via thymeleaf-extras-springsecurity.

phases:
  - phase: 1 — Permit / in SecurityConfiguration
    files:
      - src/main/java/simple/simple_webapp/SecurityConfiguration.java
        change: add "/" to permitAll requestMatchers
        from: .requestMatchers("/login", "/register").permitAll()
        to:   .requestMatchers("/", "/login", "/register").permitAll()

  - phase: 2 — HomeController
    files:
      - src/main/java/simple/simple_webapp/HomeController.java  (NEW)
        - package simple.simple_webapp
        - @Controller
        - @GetMapping("/") → return "index"

  - phase: 3 — index.html template
    files:
      - src/main/resources/templates/index.html  (NEW)
        Sections (using Thymeleaf + sec:authorize):
          IF NOT authenticated:
            - Login form (action="/login", method=POST, CSRF hidden input, username + password fields)
            - Link to /register
          IF authenticated:
            - <h1>Welcome, [[${#authentication.name}]]</h1>
            - Link to /account/change-password
            - <a sec:authorize="hasRole('ADMIN')" href="/admin/users">Admin panel</a>
            - Logout form (POST /logout)

  - phase: 4 — Tests
    files:
      - src/test/java/simple/simple_webapp/HomeControllerTests.java  (NEW)
        - @SpringBootTest(webEnvironment=MOCK), @Import(TestcontainersConfiguration.class)
        - @MockitoBean UserManagement
        - homePageAnonymousReturns200()
        - homePageAuthenticatedReturns200() (@WithMockUser)
        - homePageContainsAdminLinkForAdmin() (@WithMockUser(roles="ADMIN"))
