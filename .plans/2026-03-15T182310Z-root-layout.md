---
title: Root layout with hierarchical script extension
created: 2026-03-15T18:23:10Z
context: root-layout
description: Add layouts/root.html as the top-level layout. User/admin layouts delegate head scripts to root. Any layout can extend scripts via an extraHead slot. Bare pages (index, login, register) use root layout directly.
---

assumptions:
- No new Maven dependency needed (no Thymeleaf Layout Dialect).
- Thymeleaf head(title, extraHead) fragment selected on <head> element in root.html works correctly when called from user/admin layout via th:replace.
- xmlns:sec on <html> is cosmetic — sec: attributes work globally regardless of which file declares the namespace.
- index.html, login.html, register.html currently have no htmx/hyperscript — root layout adds them; this is acceptable overhead.
- Existing layout signatures layout(title, content) on user.html/admin.html remain unchanged (content pages need no changes).

phases:
- phase 1 — Create layouts/root.html
    file: src/main/resources/templates/layouts/root.html (NEW)
    - <html th:fragment="layout(title, extraHead, content)"> — full-page bare layout (no nav)
    - <head th:fragment="head(title, extraHead)"> — standalone head fragment
      - <meta charset="UTF-8">
      - <meta name="viewport" content="width=device-width, initial-scale=1.0">
      - <title th:replace="${title}">Page</title>
      - <script th:src="@{/webjars/htmx.org/dist/htmx.min.js}" defer>
      - <script th:src="@{/webjars/hyperscript.org/dist/_hyperscript.min.js}" defer>
      - CSRF configRequest listener (th:inline="javascript")
      - <th:block th:replace="${extraHead}"> — extension point
    - <body th:replace="${content}">Content</body>

- phase 2 — Update user.html and admin.html to delegate head to root
    file: src/main/resources/templates/layouts/user.html (EDIT)
    - Replace entire <head>...</head> with:
        <head th:replace="~{layouts/root :: head(${title}, ~{::userExtraHead})}">
          <th:block th:fragment="userExtraHead"><!-- extra scripts for user layout --></th:block>
        </head>
    - Remove: inline meta, viewport, script tags (moved to root)
    - Keep: th:fragment="layout(title, content)" on <html>, nav in body, <main th:replace="${content}">

    file: src/main/resources/templates/layouts/admin.html (EDIT)
    - Same transformation — replace <head> with root head fragment call
    - Extension fragment named "adminExtraHead"

- phase 3 — Apply root layout to index.html, login.html, register.html
    All three changes follow the same pattern:
      1. Add th:replace="~{layouts/root :: layout(~{::title}, ~{}, ~{::body})}" to <html>
      2. Strip <meta charset>, <meta viewport> from <head> (root provides them)
      3. Keep <title> in <head> (root's layout uses ~{::title})
      4. Keep all <body> content unchanged — passed as ${content} to root

    file: src/main/resources/templates/index.html (EDIT)
    - xmlns:sec stays (cosmetic, harmless)
    - Add th:replace on <html>

    file: src/main/resources/templates/login.html (EDIT)
    - Add th:replace on <html>

    file: src/main/resources/templates/register.html (EDIT)
    - Add th:replace on <html>

- phase 4 — Verify
    command: ./mvnw test
    expected: BUILD SUCCESS, 34 tests pass
    rollback: revert template edits
